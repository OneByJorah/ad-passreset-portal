# v2.1 Status Check Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an authenticated, read-only **Status Check** so a user binds with their current password and sees their *resolved* password-expiry date plus the live AD password policy, with the Status view becoming the portal's landing screen and Password Change offered as an action from it.

**Architecture:** A new `IPasswordChangeProvider.GetUserPasswordStatusAsync` method (implemented on the Windows, LDAP, and Debug providers, passed through the Lockout and LocalPolicy decorators) reads the per-user constructed attribute `msDS-UserPasswordExpiryTimeComputed` and degrades to the existing domain-default calc when unreadable. A new `POST /api/password/status` controller action reuses the existing authenticated bind, the STAB-013 enumeration redaction, the `password-fixed-window` rate-limit family, reCAPTCHA, and the SIEM audit pattern. The React app's landing screen becomes a Status view that, on a successful status check, offers the existing `PasswordForm` as the next action.

**Tech Stack:** ASP.NET Core 10 (C# 13), xUnit v3, `System.DirectoryServices.AccountManagement` (Windows) + `System.DirectoryServices.Protocols` (LDAP), React 19 / MUI 6 / Vite / Vitest + RTL.

```csharp
    /// <summary>
    /// Decodes msDS-UserPasswordExpiryTimeComputed (a Windows FILETIME). 0 and Int64.MaxValue
    /// both mean "never expires". Returns (null, false) for unparseable input. Pure — unit-testable.
    /// </summary>
    public static (DateTimeOffset? Expires, bool NeverExpires) DecodeExpiry(string? rawFileTime)
    {
        if (!long.TryParse(rawFileTime, out var raw)) return (null, false);
        if (raw == 0 || raw == long.MaxValue) return (null, true);
        try { return (DateTimeOffset.FromFileTime(raw), false); }
        catch (ArgumentOutOfRangeException) { return (null, false); }
    }

    public async Task<PasswordStatus> GetUserPasswordStatusAsync(string username, string currentPassword)
    {
        // 1) Authenticate + resolve the user's DN exactly as the change flow does. Reuse the
        //    EXISTING username-resolution search (the same one PerformPasswordChangeAsync uses
        //    to find the entry over AllowedUsernameAttributes) and an authenticating bind with
        //    the user's own credentials. On failure return the precise code; the controller redacts.
        //    Mirror the existing bind+search idiom in this file (session.Bind(); SendRequest search;
        //    GetFirstStringValueOrNull). Capture the resolved entry/DN AND, while you have it,
        //    request the PwdExpiryComputed attribute in the SAME search (AD returns constructed
        //    attributes only when explicitly named in the attribute list).
        var resolved = ResolveUserEntryWithAuth(username, currentPassword); // returns (ApiErrorCode? error, SearchResultEntry? entry)
        if (resolved.error is not null || resolved.entry is null)
            return new PasswordStatus(false, resolved.error ?? ApiErrorCode.InvalidCredentials, null, false, ExpirySource.Unknown, null);

        // 2) Decode the per-user constructed attribute from the entry we already fetched.
        var raw     = GetFirstStringValueOrNull(resolved.entry, LdapAttributeNames.PwdExpiryComputed);
        var decoded = DecodeExpiry(raw);
        var expires = decoded.Expires;
        var never   = decoded.NeverExpires;
        var source  = (raw is not null) ? ExpirySource.Resolved : ExpirySource.DomainDefault;

        // 3) Degrade: if the constructed attribute was absent, fall back to the domain-default
        //    signal. We don't compute an exact date here (that needs pwdLastSet + maxPwdAge math
        //    and is intentionally out of scope); leaving expires=null with Source=DomainDefault
        //    drives the UI caveat. If the domain has no expiry policy at all, mark never-expires.
        if (source == ExpirySource.DomainDefault && !never && expires is null)
        {
            if (GetDomainMaxPasswordAge() == TimeSpan.MaxValue) never = true;
        }

        var policy = await GetEffectivePasswordPolicyAsync();
        return new PasswordStatus(true, null, expires, never, source, policy);
    }
```

> Implementer note (real idioms in this file — do NOT invent new ones): there is NO `AuthenticateUser` or `session.ReadAttribute` method. The file's pattern is: `using var session = _sessionFactory(); session.Bind();` then a `SearchRequest`/`SendRequest` over `AllowedUsernameAttributes`, reading results with `GetFirstStringValueOrNull(entry, attr)` (see the existing `GetUserEmail`/`GetEffectivePasswordPolicyAsync` search blocks around lines 466–581). `ResolveUserEntryWithAuth` above is a NEW private helper you write by extracting/adapting the existing resolve-and-bind logic the change flow uses — it must (a) resolve the entry over the username attributes, (b) attempt an authenticating bind with `currentPassword` (return `InvalidCredentials` on bind failure, `UserNotFound` when no entry resolves), and (c) include `LdapAttributeNames.PwdExpiryComputed` in the search's requested-attributes list so the constructed attribute comes back. Reuse the change flow's bind path; do NOT add a second credential path.

```csharp
    internal static (DateTimeOffset? Expires, bool NeverExpires) DecodeExpiry(long raw)
    {
        if (raw == 0 || raw == long.MaxValue) return (null, true);
        try { return (DateTimeOffset.FromFileTime(raw), false); }
        catch (ArgumentOutOfRangeException) { return (null, false); }
    }

    public async Task<PasswordStatus> GetUserPasswordStatusAsync(string username, string currentPassword)
    {
        using var ctx = AcquirePrincipalContext();

        var user = FindUser(ctx, username);
        if (user is null)
            return new PasswordStatus(false, ApiErrorCode.UserNotFound, null, false, ExpirySource.Unknown, null);

        var upn = user.UserPrincipalName ?? username;
        if (!ValidateUserCredentials(upn, currentPassword, ctx))
            return new PasswordStatus(false, ApiErrorCode.InvalidCredentials, null, false, ExpirySource.Unknown, null);

        DateTimeOffset? expires = null;
        var never  = false;
        var source = ExpirySource.DomainDefault;
        try
        {
            var entry = (DirectoryEntry)user.GetUnderlyingObject();
            entry.RefreshCache(new[] { "msDS-UserPasswordExpiryTimeComputed" });
            var val = entry.Properties["msDS-UserPasswordExpiryTimeComputed"].Value;
            if (val is not null)
            {
                var raw = Convert.ToInt64(val, System.Globalization.CultureInfo.InvariantCulture);
                (expires, never) = DecodeExpiry(raw);
                source = ExpirySource.Resolved;
            }
        }
        catch (Exception ex) when (ex is System.Runtime.InteropServices.COMException or System.DirectoryServices.DirectoryServicesCOMException)
        {
            _logger.LogWarning(ex, "GetUserPasswordStatusAsync: per-user expiry read failed; degrading to domain default");
            source = ExpirySource.DomainDefault;
        }

        if (source == ExpirySource.DomainDefault && !never && expires is null)
        {
            if (GetDomainMaxPasswordAge() == TimeSpan.MaxValue) never = true;
        }

        var policy = await GetEffectivePasswordPolicyAsync();
        return new PasswordStatus(true, null, expires, never, source, policy);
    }
```

```csharp
    /// Status Check (v2.1): authenticate with the current password and return resolved

    [ProducesResponseType(StatusCodes.Status200OK)]
    [ProducesResponseType(StatusCodes.Status400BadRequest)]

    public async Task<IActionResult> StatusAsync([FromBody] StatusCheckModel model)
    {
        var clientIp = HttpContext.Connection.RemoteIpAddress?.ToString() ?? "unknown";

        if (!ModelState.IsValid)
        {
            Audit("StatusValidationFailed", model.Username, clientIp, SiemEventType.ValidationFailed);
            return BadRequest(ApiResult.FromModelStateErrors(ModelState));
```
